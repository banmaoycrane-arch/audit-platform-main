# Tasks

## 核心问题：验证码登录用户无密码

### 问题分析
- 验证码登录时只创建 phone 用户，不设置 hashed_password
- 密码登录需要 hashed_password 存在才能工作
- 当前没有在登录后引导用户设置密码

### 解决方案

- [x] Task 1: 改进登录后引导——设置密码入口
  - 用户通过验证码登录成功后
  - 如果账号没有设置密码
  - 弹窗提示"设置密码后可以使用账号密码登录"
  - 提供设置密码表单（密码 + 确认密码）
  - 设置成功后，下次可直接用密码登录

- [x] Task 2: 改造 `TeamManagementPage` 团队列表为卡片网格形式
  - 显示：团队名称、类型、成员数量、账簿数量、创建时间
  - 支持点击选中高亮
  - 右上角操作菜单（编辑/删除/人员管理）
  - 空状态提示和创建按钮

- [x] Task 3: 改造 `LedgerManagementPage` 账簿列表为卡片网格形式
  - 显示：账簿名称、所属团队、状态、创建时间
  - 不同状态用不同边框颜色区分
  - 快捷操作按钮（切换/生命周期/授权）
  - 空状态提示和创建按钮
  - 支持状态筛选

- [x] Task 4: 改造 `ProjectsPage` 项目列表为卡片网格形式
  - 显示：项目名称、类型、状态、团队、负责人、日期
  - 不同状态用不同颜色标识
  - 操作菜单
  - 空状态提示和创建按钮
  - 支持状态筛选

## Task Dependencies
- Task 1（登录后引导）是核心功能，优先完成
- Task 2, 3, 4 可并行开发
