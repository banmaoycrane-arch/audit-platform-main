# 团队与账套管理界面补充 Spec

## Why

账套、团队、项目等核心概念已在后端建立，但前端缺少对应的管理界面。用户登录后：
- 无法创建或管理团队
- 无法创建或管理账套
- 无法查看或修改用户授权
- 登录流程没有引导用户完成"团队 → 账套"的初始化链路

需要补充完整的管理界面，让概念真正可用。

## What Changes

- 新增团队管理页面：创建团队、查看团队列表、管理团队人员
- 新增账套管理页面：创建账套、查看账套列表、管理账套生命周期、管理账套授权
- 新增用户授权管理：查看授权列表、授予/撤销授权
- 改造登录后流程：首次登录引导创建团队 → 创建账套 → 进入工作台
- 在现有工作台增加管理入口

## Impact

- Affected specs:
  - `team-multi-ledger-management`
  - `ledger-register-project-concept-unification`
  - `lifecycle-management`
- Affected code:
  - `frontend/src/pages/` — 新增 TeamManagementPage, LedgerManagementPage
  - `frontend/src/api/client.ts` — 新增团队/账套管理 API
  - `frontend/src/App.tsx` — 新增路由
  - `frontend/src/pages/LoginPage.tsx` — 改造登录后引导流程

## ADDED Requirements

### Requirement: 团队管理页面

系统 SHALL 提供团队管理界面。

#### Scenario: 创建团队
- **WHEN** 用户点击"创建团队"
- **THEN** 弹出表单：团队名称、团队类型（事务所/企业/虚拟）
- **AND** 创建后用户自动成为团队管理员

#### Scenario: 团队列表
- **WHEN** 用户进入团队管理页面
- **THEN** 显示用户所属的所有团队
- **AND** 显示团队名称、类型、成员数量、账套数量

#### Scenario: 管理团队人员
- **WHEN** 用户点击团队"人员管理"
- **THEN** 显示团队成员列表
- **AND** 支持添加成员（通过用户名/手机号）、移除成员、修改角色

### Requirement: 账套管理页面

系统 SHALL 提供账套管理界面。

#### Scenario: 创建账套
- **WHEN** 用户在团队下点击"创建账套"
- **THEN** 弹出表单：账套名称、所属团队（默认当前团队）
- **AND** 创建后用户自动获得 admin 权限

#### Scenario: 账套列表
- **WHEN** 用户进入账套管理页面
- **THEN** 显示用户有权限的所有账套
- **AND** 显示账套名称、所属团队、状态、创建时间

#### Scenario: 账套生命周期管理
- **WHEN** 用户点击账套"生命周期"
- **THEN** 显示当前状态（草稿/启用/暂停/归档）
- **AND** 支持状态转换操作（激活、暂停、归档、恢复）
- **AND** 操作时需要填写原因

#### Scenario: 账套授权管理
- **WHEN** 用户点击账套"授权管理"
- **THEN** 显示已授权用户列表
- **AND** 支持添加授权（选择用户、设置角色）、撤销授权

### Requirement: 登录后引导流程

系统 SHALL 在首次登录时引导用户完成初始化。

#### Scenario: 首次登录无团队
- **WHEN** 用户首次登录且没有团队
- **THEN** 显示引导页面："欢迎使用，请先创建团队"
- **AND** 提供创建团队表单
- **AND** 创建后引导创建账套
- **AND** 完成后进入工作台

#### Scenario: 已有团队无账套
- **WHEN** 用户登录有团队但没有账套
- **THEN** 显示引导页面："请创建或选择账套"
- **AND** 提供创建账套表单或选择现有账套

#### Scenario: 已有团队已有账套
- **WHEN** 用户登录已有团队和账套
- **THEN** 直接进入工作台
- **AND** 默认进入 last_ledger_id 对应的账套

### Requirement: 工作台管理入口

系统 SHALL 在工作台提供管理入口。

#### Scenario: 顶部管理菜单
- **WHEN** 用户在工作台顶部
- **THEN** 用户下拉菜单增加：
  - "团队管理"
  - "账套管理"
  - "项目管理"
- **AND** 点击后跳转到对应管理页面

## MODIFIED Requirements

### Requirement: 登录页面

登录成功后增加引导逻辑：
- 检查用户是否有团队
- 检查用户是否有账套
- 根据情况引导到不同页面

### Requirement: 工作台页面

顶部用户下拉菜单增加管理入口。

## REMOVED Requirements

无。

## 财务视角说明

- **团队 = 事务所/企业**：会计师不是独立工作，而是属于某个组织。团队管理是"组织管理"的基础。
- **账套 = 客户账簿**：一个团队服务多个客户，每个客户一套账。账套管理是"客户管理"的基础。
- **授权 = 项目权限**：不是所有人都能看所有客户的账，而是根据项目分配。授权管理是"内控"的基础。
- **引导流程 = 入职流程**：新员工首次登录，需要创建团队（或加入团队），然后创建账套（或获得授权），最后才能开始工作。这是财务系统的标准入职流程。
- **为什么需要管理界面**：没有管理界面，概念只是数据库里的记录，用户无法操作。管理界面让概念真正可用。
