# Trae 自定义指令：核心主线守护

当前项目阶段：核心记账闭环 + 导入解析引擎。

## 每次执行任务前必须执行

1. 优先读取 `.trae/daily-focus.md` 和 `.trae/weekly-focus.md`。
2. 判断用户请求属于以下哪一类：
   - **CORE-CLOSURE**（核心闭环）：直接属于核心记账闭环或导入解析引擎 → 立即执行
   - **CORE-SUPPORT**（核心支持）：为核心模块提供必要支持 → 次优先执行
   - **INFRA**（基础设施）：环境、配置、构建 → 仅在阻塞核心时执行
   - **NICE-TO-HAVE**（锦上添花）：优化、美化、非核心扩展 → 必须提醒用户并建议延期
   - **DRIFT**（偏离）：与核心主线无关 → 必须拒绝或转交 backlog
3. 执行前用一句话说明任务属于哪一类。
4. 执行完成后更新 `.trae/daily-focus.md` 的完成状态。

## 禁止行为

- 不主动建议非核心功能开发
- 不做大规模架构重构或重命名
- 不创建与核心模块无关的新页面、新 API、新服务

## 提醒话术模板

当用户请求偏离核心主线时：

> “当前项目阶段聚焦于【核心记账闭环】和【导入解析引擎】。该请求属于 [NICE-TO-HAVE/DRIFT]，建议记录到 backlog，待核心闭环稳定后再处理。是否继续？”
