# 财务验收测试清单

> 适用版本：Phase 1-3 完成后  
> 测试地址：http://127.0.0.1:5173（前端） / http://127.0.0.1:8010（后端）  
> 测试角色：专业会计师 / 审计人员

---

## 使用前准备

1. 启动后端：`source .venv/bin/activate && pnpm dev:backend`
2. 启动前端：`pnpm dev:frontend`
3. 准备测试文件：
   - 一份 Excel/CSV 会计分录（含凭证号、日期、摘要、科目、借贷金额）
   - 一份 PDF/TXT 原始资料（可选：发票、合同、银行流水）
4. 建议使用独立测试账号，避免污染正式账套数据

## 九、问题记录模板

发现问题时请记录：

```text
【页面/路径】：
【操作步骤】：
【预期结果】：
【实际结果】：
【财务影响】：（如：报表不平衡、审计结论错误、权限越界）
【优先级】：高 / 中 / 低
【截图/附件】：
```

**优先级规则**：
- **高**：财务逻辑错误、借贷不平、权限漏洞、数据丢失
- **中**：流程卡顿、字段缺失、导出格式问题
- **低**：界面美观、文案、非阻断体验问题

## 十、建议验收顺序

```text
登录注册 → 建团队/账套/项目 → 基础资料 → 记账全流程 → 损益结转+报表
    → 审计全流程 → 银行对账 → 工作台总览
```

完成一轮后，将「通过」列改为 ✅，未通过项反馈给开发优先修复。
