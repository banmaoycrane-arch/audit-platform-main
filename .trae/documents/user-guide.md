# 财务向量审计系统 - 使用说明

## 一、系统概述

财务向量审计系统是一个面向会计师和审计师的专业财务核算与审计工具，支持：

- **记账主线**：凭证录入/导入 → 复核 → 结账 → 报表
- **审计主线**：审计计划 → 证据导入 → 测试 → 发现 → 报告
- **AI 辅助**：智能分录生成、风险识别、证据充分性判断

---

## 二、登录与注册

### 2.1 登录方式

系统支持两种登录方式：

| 方式 | 适用场景 | 特点 |
|---|---|---|
| **账号密码登录** | 长期使用、安全性要求高 | 需要预先注册并设置密码 |
| **验证码登录** | 临时登录、快速验证 | 自动创建账号（首次使用） |

### 2.2 登录步骤

1. 打开浏览器访问：http://127.0.0.1:5173
2. 选择登录方式：
   - **账号密码登录**：输入用户名/手机号和密码，点击"登录"
   - **验证码登录**：输入手机号，点击"获取验证码"（开发环境默认验证码：123456），输入验证码后点击"登录"

### 2.3 注册步骤

1. 点击登录页面的"立即注册"链接
2. 填写用户名、手机号、密码
3. 勾选同意用户协议和隐私政策
4. 点击"注册"完成注册

### 2.4 忘记密码

1. 在账号密码登录页面点击"忘记密码？"链接
2. 输入手机号获取验证码
3. 验证通过后设置新密码

> **注意**：通过验证码登录的用户默认没有设置密码，需要先使用验证码登录后在个人中心设置密码。

## 三、首次登录引导

首次登录系统时，系统会引导您完成以下步骤：

1. **创建团队**：团队是组织的最高层级，用于隔离不同组织的数据
2. **创建账套**：账套是会计核算的核心单位，每个账套独立管理科目、凭证和报表
3. **创建项目**：项目用于管理具体的审计或核算任务
4. **确认会计主体**：设置账套对应的会计主体信息

完成引导后，系统会自动跳转到工作台首页。

## 四、记账流程

### 4.1 入口

从侧边栏菜单进入「记账」模块，或直接访问 `/accounting/step/1`

### 4.2 流程步骤

| 步骤 | 名称 | 功能 |
|---|---|---|
| Step1 | 选择类型 | 选择账套、会计期间、资料来源类型 |
| Step2 | 导入资料 | 上传原始资料文件（Excel/CSV）或使用 AI 生成凭证草稿 |
| Step3 | 生成分录 | 系统自动生成会计分录草稿 |
| Step4 | 复核调整 | 人工复核分录，可修改或驳回 |
| Step5 | 确认入账 | 凭证落库，支持导出 Excel/JSON |

### 4.3 操作提示

- **原始资料格式**：支持 CSV、Excel 格式，建议包含：凭证号、日期、摘要、科目、借方金额、贷方金额
- **AI 路径**：选择 AI 生成方式时，系统会自动识别资料内容并推荐会计科目
- **分录平衡**：系统会自动检查借贷平衡，不平衡的凭证无法提交

## 五、审计流程

### 5.1 入口

从侧边栏菜单进入「审计」模块，或直接访问 `/audit/step/1`

### 5.2 流程步骤

| 步骤 | 名称 | 功能 |
|---|---|---|
| Step1 | 选择范围 | 创建审计项目，选择审计期间和范围 |
| Step2 | 上传证据 | 上传审计证据文件（合同、发票、序时簿等） |
| Step3 | 导入分录 | 导入被审计单位序时簿，生成审计分录 |
| Step4 | 执行测试 | 执行内控测试、业务循环测试 |
| Step5 | 复核发现 | 记录审计发现，评估风险等级 |
| Step6 | 导出报告 | 生成并导出审计报告 |

### 5.3 审计测试类型

- **内控测试**：穿行测试、控制测试、问卷测试
- **业务循环测试**：采购循环、销售循环、资金循环
- **实质性测试**：抽样测试、分析性复核

## 六、基础资料管理

### 6.1 会计科目

- **功能**：维护会计科目表，支持科目新增、修改、删除
- **入口**：侧边栏 → 基础资料 → 会计科目
- **特点**：
  - 支持多级科目结构
  - 预设行业模板（如企业会计准则、小企业会计准则）
  - 科目方向（借方/贷方）影响报表取数

### 6.2 往来单位

- **功能**：管理客户、供应商等往来单位信息
- **入口**：侧边栏 → 基础资料 → 往来单位
- **字段**：名称、统一信用代码、是否关联方、默认会计主体

### 6.3 期初余额

- **功能**：录入或导入期初余额
- **入口**：侧边栏 → 基础资料 → 期初余额
- **要求**：借贷必须平衡

## 七、会计期间管理

### 7.1 期间状态

| 状态 | 说明 | 允许操作 |
|---|---|---|
| Open（开放） | 正常状态 | 录入凭证、修改凭证 |
| PL Transferred（损益结转） | 已执行损益结转 | 可反结转 |
| Closed（已结账） | 已结账 | 不可修改，需反结账 |

### 7.2 操作流程

1. **创建期间**：设置期间编码、起止日期
2. **录入凭证**：在开放期间内录入凭证
3. **损益结转**：将损益类科目余额转入本年利润
4. **结账**：锁定期间，禁止修改
5. **反结账**：解除锁定（谨慎使用）

## 八、报表查询

### 8.1 试算平衡表

- 展示所有科目的期初余额、本期发生额、期末余额
- 自动检查借贷平衡

### 8.2 资产负债表

- 资产 = 负债 + 所有者权益
- 按会计期间查询

### 8.3 利润表

- 收入 - 费用 = 净利润
- 与损益结转数据一致

---

## 九、AI 功能

### 9.1 智能分录生成

- 根据原始资料内容，自动推荐会计科目
- 支持人工复核和调整
- 记录 AI 建议与人工修改的差异日志

### 9.2 风险识别

- 自动分析序时簿数据
- 识别异常交易、大额波动、关联方交易等风险点
- 输出风险线索供审计人员关注

### 9.3 证据充分性判断

- AI 评估审计证据是否充分
- 证据不足时给出风险提示
- 不替代人工判断

## 十三、常见问题

### 13.1 密码登录无响应

**问题**：账号密码登录点击后无反应

**原因**：通过验证码登录的用户尚未设置密码

**解决**：
1. 使用验证码登录系统
2. 在个人中心设置密码
3. 之后即可使用密码登录

### 13.2 凭证无法落库

**检查项**：
- 借贷金额是否平衡
- 会计期间是否已结账
- 科目代码是否存在

### 13.3 报表不平衡

**排查步骤**：
1. 检查期初余额是否平衡
2. 检查本期凭证是否全部平衡
3. 检查损益结转是否执行
4. 核对报表取数逻辑
