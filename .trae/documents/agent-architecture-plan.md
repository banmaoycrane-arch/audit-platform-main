# Agent 架构设计规划

## 1. 当前架构分析

### 1.1 现有架构

```
前端（React）                    后端（FastAPI）
┌──────────────┐               ┌──────────────┐
│  Layout      │               │  API Routes  │
│  - 仪表盘    │  ←  API →    │  - imports   │
│  - 导入      │               │  - entries   │
│  - 分录      │               │  - files     │
│  - 风险      │               │  - risks     │
└──────────────┘               └──────────────┘
        ↓
   人工选择模式
   (记账/审计)
```

### 1.2 当前设计的问题

| 问题 | 说明 |
|-----|------|
| 模式选择依赖人工 | 用户需要自己判断用记账还是审计模式 |
| 流程固化 | 5步/6步引导是固定的 |
| 无法对话式交互 | 用户不能通过自然语言下达任务 |
| 缺乏智能决策 | 系统不能根据上下文自动选择模式 |

## 2. Agent 架构设计

### 2.1 目标架构

```
用户（自然语言）
        ↓
┌──────────────────────────────────────┐
│           Agent 入口层                │
│  ┌────────────────────────────────┐  │
│  │   Intent Recognition           │  │
│  │   意图识别                      │  │
│  │   - 记账意图                    │  │
│  │   - 审计意图                    │  │
│  │   - 查询意图                    │  │
│  └────────────────────────────────┘  │
│                 ↓                   │
│  ┌────────────────────────────────┐  │
│  │   Task Planning                │  │
│  │   任务规划                      │  │
│  │   - 选择执行模式                 │  │
│  │   - 分解子任务                  │  │
│  │   - 编排执行顺序                │  │
│  └────────────────────────────────┘  │
│                 ↓                   │
│  ┌────────────────────────────────┐  │
│  │   Execution Engine             │  │
│  │   执行引擎                      │  │
│  │   - 调用后端 API               │  │
│  │   - 处理结果                    │  │
│  │   - 生成回复                    │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
        ↓
┌──────────────┐               ┌──────────────┐
│  前端 UI     │               │  后端服务    │
│  (Agent Chat)│               │  (现有 API) │
└──────────────┘               └──────────────┘
```

### 2.2 与现有设计的兼容性

| 现有组件 | 兼容性 | 说明 |
|---------|-------|-----|
| 记账模式（5步） | ✅ 兼容 | Agent 自动执行，用户可旁观/干预 |
| 审计模式（6步） | ✅ 兼容 | Agent 自动执行，用户可旁观/干预 |
| 原始资料导入 | ✅ 兼容 | 作为 Agent 执行的子任务 |
| 审计测试服务 | ✅ 兼容 | Agent 调用并整合结果 |
| 审计报告生成 | ✅ 兼容 | Agent 生成自然语言报告 |

**兼容性结论**：现有设计可以完全复用，Agent 是入口层的升级，不影响现有业务逻辑。

## 3. Agent 设计细节

### 3.1 意图识别

```python
INTENT_PATTERNS = {
    "记账意图": [
        "帮我做账", "录入凭证", "编制分录", "做凭证",
        "根据发票生成凭证", "根据银行流水做账",
    ],
    "审计意图": [
        "审计", "审查", "检查账目", "审计测试",
        "完整性测试", "截止性测试", "准确性测试",
    ],
    "查询意图": [
        "查看", "查询", "统计", "导出",
        "有多少", "看看", "显示",
    ],
    "报告意图": [
        "生成报告", "导出报告", "审计报告",
        "总结", "汇总",
    ],
}
```

### 3.2 Agent 对话示例

| 用户输入 | 识别意图 | Agent 行为 |
|---------|---------|-----------|
| "帮我根据这些发票做账" | 记账意图 | 启动记账模式流程 |
| "帮我审计一下这个月的凭证" | 审计意图 | 启动审计模式流程 |
| "哪些凭证缺少原始资料？" | 查询意图 | 查询并展示结果 |
| "生成一份审计报告" | 报告意图 | 生成并返回报告 |

### 3.3 Agent 与模式的对应关系

```
用户意图 → Agent 决策 → 执行模式

"做账" → 记账模式 → 原始资料导入 → AI编制定录 → 凭证复核 → 导出

"审计" → 审计模式 → 原始资料导入 → 分录导入 → 审计测试 → 生成报告
```

### 4.3 消息格式

```typescript
interface AgentMessage {
  id: string
  role: 'user' | 'agent' | 'system'
  content: string
  attachments?: {
    type: 'image' | 'file'
    url: string
  }[]
}

interface AgentTask {
  task_type: 'accounting' | 'audit' | 'query' | 'report'
  status: 'pending' | 'running' | 'completed' | 'failed'
  progress: number
  steps: AgentTaskStep[]
  result?: any
}

interface AgentTaskStep {
  name: string
  status: 'pending' | 'running' | 'completed' | 'failed'
  result?: any
}
```

## 5. 实施计划

### Phase 1: Agent 入口层（优先级：高）
1. 创建 Agent Chat 界面
2. 实现基础对话功能
3. 实现意图识别

### Phase 2: 任务编排层（优先级：高）
1. 实现任务规划器
2. 对接现有记账/审计模式
3. 实现进度跟踪

### Phase 3: 智能增强（优先级：中）
1. 接入大模型（LLM）
2. 自然语言生成报告
3. 智能问答

---

## 6. 架构演进路线

```
Phase 1: 简单规则匹配
用户: "做账" → 规则匹配 → 执行记账模式

Phase 2: LLM 增强
用户: "帮我看看这个月有哪些异常" → LLM 理解 → 执行查询+分析 → 自然语言回复

Phase 3: 完全自主
用户: "对这个企业做个全面审计" → LLM 自主规划 → 执行所有测试 → 生成完整报告
```

## 7. 结论

### 7.1 兼容性评估

| 方面 | 评估 |
|-----|------|
| 现有记账模式 | ✅ 完全兼容 |
| 现有审计模式 | ✅ 完全兼容 |
| 现有 API | ✅ 完全兼容 |
| 现有数据模型 | ✅ 完全兼容 |

### 7.2 风险评估

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| LLM 理解偏差 | 中 | 规则兜底 + 确认机制 |
| 执行时间不确定 | 中 | 进度展示 + 可中断 |
| 多轮对话状态管理 | 低 | 会话持久化 |

### 7.3 建议

1. **短期**：先实现 Phase 1（规则匹配），快速上线
2. **中期**：接入 LLM，提升体验
3. **长期**：实现完全自主的 Agent
