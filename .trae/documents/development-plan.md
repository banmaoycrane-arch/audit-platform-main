# 财务向量审计系统 MVP 开发计划

## 一、项目目标

构建「从账簿数据导入 → 记账闭环 → 审计风险识别 → 工作底稿」的一体化工具。

当前阶段聚焦两条核心主线：
- **文件智能解析**：双引擎对比 + 修正回流闭环 + embedding 字段对齐（静态别名为主，embedding 兜底）。
- **完整财务记账闭环**：项目 → 账簿 → 会计分录 → 凭证 → 期间 → 报表。

## 二、技术栈

- 后端：Python + FastAPI + SQLAlchemy + SQLite（默认）/ PostgreSQL（可选）
- 前端：React + Vite + Ant Design + TypeScript
- 向量存储：Qdrant
- 大模型：局域网 Ollama（`http://192.168.2.8:11434/v1`），模型 `nomic-embed-text` / `qwen2.5:7b`
- 包管理：pnpm（前端）、pip + venv（后端）

## 三、阶段划分

### P1：基础边界与阻塞修复（1-2 天）

**目标**：修复已知阻塞问题，让核心服务稳定可启动，测试可跑通。

**任务清单**：
1. 修复 `embedding_service.py` 对 API Key 的强依赖，支持 Ollama 无 key 调用。
2. 修复 `embedding_service.py` 异步事件循环冲突，改用同步 `httpx.Client`。
3. 补充 `backend/.env` 中的 Ollama 配置。
4. 修复前期遗留的 `test_parser_engine_analyzer.py` 单元测试失败：
   - `test_normalize_comparison_value` 数值格式问题
   - `test_extract_numeric` Decimal/float 比较问题
   - `test_analyze_dual_engine_result_consistent` 字段数量断言问题
5. 扩充 `_FIELD_NAME_ALIASES` 静态别名表，覆盖 POC 中测试的常用同义字段。
6. 清理临时验证脚本，跑通全部相关测试。

**验收标准**：
- `pytest tests/test_field_embedding_aligner.py -v` 通过
- `pytest tests/test_parser_engine_analyzer.py -v` 通过
- 前后端可正常启动（`pnpm dev:backend`、`pnpm dev:frontend`）

### P2：文件智能解析稳定性提升（3-5 天）

**目标**：让文件解析综合稳定性达到 **96%**（阶段指标）。

**任务清单**：
1. 完善修正回流闭环：
   - 人工复核界面可记录正确字段与值
   - 从修正记录提取规则并写入 `ParseCorrectionRule`
   - 下次解析时自动应用修正规则
2. 强化字段标准化：
   - 静态别名表按文档类型分类（合同、发票、银行流水）
   - embedding 对齐仅作为静态表未命中时的兜底策略
3. 提升双引擎一致性：
   - 对低一致性字段自动触发人工复核
   - 记录不一致模式，用于后续规则优化
4. 增加解析稳定性指标看板：
   - 字段准确率
   - 文档完整率
   - 引擎一致性率
   - 修正回流有效率

**验收标准**：
- 使用标准测试样本集，综合稳定性指标 ≥ 96%
- 修正记录可正确回流并影响下一次解析
- 双引擎低一致性结果自动标记复核

### P3：财务记账闭环完善（5-7 天）

**目标**：实现从数据导入到报表生成的完整记账闭环。

**任务清单**：
1. 账簿与会计期间管理：
   - 期间状态机：open → pl_transferred → closed
   - 已结账期间修改需审计日志与权限控制
2. 凭证与分录：
   - 凭证生成、审核、过账
   - 借贷平衡强制校验
   - 支持从解析结果自动生成凭证草稿
3. 损益结转：
   - 自动损益结转凭证生成
   - 结转后资产负债表平衡校验
4. 报表输出：
   - 资产负债表
   - 利润表
   - 现金流量表（简化版）

**验收标准**：
- 可完成一个会计期间的完整闭环：录入 → 凭证 → 过账 → 损益结转 → 结账 → 报表
- 报表勾稽关系正确
- 已结账期间不可随意修改

### P4：审计主线打通（3-5 天）

**目标**：实现审计任务 → 业务循环 → 风险识别 → 工作底稿 → 审计发现的最小闭环。

**任务清单**：
1. 审计任务与范围：
   - 审计任务绑定 `ledger_id`
   - 按业务循环选择审计范围
2. 风险识别：
   - 基于规则的风险扫描
   - 基于向量相似度的异常检测（可选，后续增强）
3. 工作底稿：
   - 底稿创建、编辑、版本管理
   - 关联审计证据和调整分录
4. 审计发现：
   - 风险等级、事实描述、影响、建议

**验收标准**：
- 可创建一个审计任务并完成一次最小循环
- 风险识别能命中已知异常模式
- 工作底稿可保存并导出

### P5：集成测试与真实数据验证（2-3 天）

**目标**：确保财务逻辑零错误，主流程可演示。

**任务清单**：
1. 编写端到端测试覆盖主流程
2. 使用真实样本数据验证解析稳定性
3. 验证报表勾稽关系
4. 整理演示脚本和操作手册

**验收标准**：
- 财务逻辑错误数为 0
- 真实样本解析稳定性 ≥ 96%
- 主流程可完整演示

## 五、质量保障措施

1. **单元测试**：每个核心函数配套测试，财务计算必须覆盖边界值。
2. **财务规则校验**：借贷平衡、期间状态机、报表勾稽关系强制校验。
3. **代码审查**：遵循 AGENTS.md 中的命名、注释、精度规范。
4. **阶段验收**：每阶段完成后列出测试地址和验证方法，用户确认后再进入下一阶段。
5. **问题升级**：财务逻辑错误为最高优先级 bug，立即修复。
