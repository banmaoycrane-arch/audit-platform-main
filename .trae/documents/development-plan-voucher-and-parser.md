# 项目开发规划：优先解决基础功能堵点

> **文档类型**: 项目开发执行规划
> **更新日期**: 2026-07-01
> **规划目标**: 优先解决当前基础功能堵点，完成记账凭证管理系统和文件解析引擎两大核心模块，确保核心业务流程闭环可用
> **执行原则**: 先打通堵点 → 再完善体验 → 最后扩展辅助功能

---

## 一、项目当前背景

### 1.1 已具备基础

- 数据模型已定义：`Voucher` 主表、`AccountingEntry` 分录表、`Ledger` 账簿表等核心模型已落地
- 后端服务已有 `voucher_service.py` 提供凭证创建、借贷平衡校验等基础能力
- 文件解析已有 `file_parser_service.py` 支持 Excel 等文件解析
- 前端路由已具备 `/entries`、`/ledger/entries` 等入口
- 审计工作流模型（AuditTask/WorkBranch/ReviewRequest）已定义

### 1.2 当前核心堵点

1. **记账凭证管理功能不完整**：有 `Voucher` 模型，但缺少完整的前后端生命周期管理（查看、编辑、选择、状态流转）
2. **文件解析引擎稳定性不足**：解析逻辑存在多种文件格式适配、表头检测、金额推断等问题，影响数据导入准确性
3. **核心业务流程未完全闭环**：从文件导入到凭证生成、查看、编辑、过账的链路存在断点

---

## 二、开发目标

### 2.1 总体目标

在 4~6 周内完成两大核心模块的基础功能闭环：

1. **记账凭证管理系统**：实现凭证的完整生命周期管理
2. **文件解析引擎**：提升稳定性，准确高效处理各类财务文件

### 2.2 具体目标

| 模块 | 目标 |
|------|------|
| 记账凭证管理系统 | 凭证录入、凭证查看、凭证选择、凭证编辑，全流程支持借贷平衡校验 |
| 文件解析引擎 | 支持常见财务文件格式（Excel/CSV），自动表头检测、字段映射、金额方向推断，错误提示清晰 |
| 业务闭环 | 文件导入 → 解析 → 生成凭证草稿 → 查看编辑 → 校验过账 → 查询管理 |

### 4.2 关键流程设计

#### 4.2.1 凭证生命周期状态机

```
┌─────────┐    保存      ┌──────────┐    复核      ┌─────────┐    过账      ┌──────────┐
│  草稿   │ ─────────> │  已复核   │ ─────────> │  已过账  │ ─────────> │  已结账  │
│ draft   │            │ verified │            │ posted  │            │ closed  │
└─────────┘            └──────────┘            └─────────┘            └──────────┘
     │                      │                      │                      │
     │ 删除                 │ 退回                 │ 作废                 │  -     │
     ▼                      ▼                      ▼                      ▼
┌─────────┐            ┌──────────┐            ┌─────────┐            ┌──────────┐
│  已删除  │            │  退回草稿 │            │  已取消  │            │   已关账  │
│ deleted │            │  draft   │            │cancelled│            │  closed  │
└─────────┘            └──────────┘            └─────────┘            └──────────┘
```

#### 4.2.2 凭证编辑时的借贷平衡校验

1. 用户修改分录行时，前端实时计算借贷合计
2. 保存时后端重新校验所有分录行
3. 不平衡时返回差额，提示用户调整
4. 支持"自动找平"功能：将差额自动填入最后一行的对方科目

#### 4.2.3 核心 API 设计

```python
# 凭证列表查询
GET /api/vouchers?ledger_id=1&period_id=2&status=draft&keyword=记-001

# 凭证详情
GET /api/vouchers/{voucher_id}

# 创建凭证
POST /api/vouchers
{
  "ledger_id": 1,
  "voucher_date": "2024-01-15",
  "summary": "支付办公费",
  "lines": [
    {"account_code": "6602", "debit_amount": 1000, "summary": "办公费"},
    {"account_code": "1002", "credit_amount": 1000, "summary": "银行存款"}
  ]
}

# 编辑凭证
PUT /api/vouchers/{voucher_id}
{
  "summary": "支付办公费（调整）",
  "lines": [...]
}

# 状态流转
POST /api/vouchers/{voucher_id}/verify
POST /api/vouchers/{voucher_id}/post
POST /api/vouchers/{voucher_id}/cancel
POST /api/vouchers/{voucher_id}/delete
```

### 5.2 关键流程设计

#### 5.2.1 文件解析流程

```
用户上传文件
    ↓
检测文件格式（Excel/CSV）
    ↓
检测表头行
    ↓
字段映射（标准字段 ↔ 文件表头）
    ↓
逐行解析（金额、日期、方向）
    ↓
行级校验（必填字段、金额非负、借贷平衡）
    ↓
按凭证号/日期分组生成候选凭证
    ↓
返回解析结果 + 错误明细 + 质量评分
    ↓
用户预览确认 → 生成凭证草稿 / 编辑后保存
```

#### 5.2.2 解析错误分类

| 错误类型 | 说明 | 处理方式 |
|---------|------|---------|
| 表头未识别 | 无法找到凭证号、摘要、金额等关键字段 | 提示用户手动映射 |
| 金额格式错误 | 无法解析为数字 | 高亮错误行，提示修正 |
| 借贷不平衡 | 同一凭证号下借贷合计不等 | 提示差额，支持拆分或补行 |
| 日期格式错误 | 无法解析日期 | 提示用户选择日期格式 |
| 科目不存在 | 科目代码在当前账簿不存在 | 提示匹配或新增科目 |
| 必填字段缺失 | 缺少凭证号、摘要等 | 提示补全 |

### 6.1 目标流程

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────────┐     ┌─────────────────┐

                                                                             ▼
                                                                    ┌─────────────────┐

                                                                    │  draft → posted │
                                                                    └─────────────────┘
                                                                             │
                                                                             ▼
                                                                    ┌─────────────────┐
                                                                    │   凭证查询/管理  │
                                                                    │ 列表/详情/筛选   │
```

### 6.2 关键集成点

| 集成点 | 说明 | 涉及模块 |
|--------|------|---------|
| 导入任务 → 解析引擎 | 上传文件后调用解析引擎 | 文件解析引擎 |
| 解析结果 → 凭证草稿 | 按凭证号分组生成候选凭证 | 记账凭证管理 |
| 凭证草稿 → 编辑页 | 用户可查看、修改、确认 | 前端页面 |
| 编辑页 → 保存/过账 | 校验后保存或执行状态流转 | 后端 API |
| 已过账 → 账簿/报表 | 更新科目余额、生成总账/明细账 | 账务处理模块 |
